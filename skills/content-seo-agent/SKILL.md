# Content and SEO agent

Make pages clear to people, crawlers, and social previews without manufacturing claims or keyword clutter. Start with page intent, then make metadata and structure match it.

## When to use this skill

Use for page launches, metadata cleanup, share-preview issues, content hierarchy, structured data, or crawlability review.

## What changes

| Before | After |
|---|---|
| Generic meta copy | Page-specific title and description |
| Visual-only hierarchy | Readable heading and link structure |
| Unsupported schema | Visible-content-backed structured data |

## Example requests

- "Make this product page ready for launch and social sharing."
- "Audit our metadata and canonical URLs."
- "Add FAQ schema only where the content supports it."

Improve discoverability without turning a page into keyword filler. Clarify search intent and page purpose. Preserve accessible reading order and write claims that the product can support.

## Workflow

1. Identify page audience, task, primary topic, conversion goal, and canonical URL.
2. Build a heading outline that lets a reader understand the page without visual styling.
3. Write title and description from actual page value, not generic marketing phrases.
4. Check canonical, robots, sitemap, language metadata, internal links, Open Graph, and share-image dimensions.
5. Add structured data only when visible page content supports every property.
6. Review image alt text, link labels, and duplicate route variants for both people and crawlers.

## Guardrails

Never invent reviews, FAQs, prices, availability, authorship, or structured-data facts. Do not hide keyword blocks, generate doorway pages, or change a canonical without understanding routing. Treat SEO recommendations as product-content decisions, not magic ranking switches.

## Handoff

Return intent, metadata values, changed files, structured-data type, validation method, and content gaps that require a human owner.
